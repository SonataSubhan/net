Cisco İmtahan Bələdçisi — Düzəlişli Prompt

Aşağıdaki tam prompt-u "Copy" düyməsi ilə kopyala və yeni chat-a yapışdır — iki HTML faylındakı (port aralığı, router/switch komandalarının qarışması, ICMP ACL nümunəsi) problemləri aradan qaldıran tələblərlə.

Bu prompt nəyi düzəldir

  • Port aralıqları — "Fa0/1-4" formatı yoxdur, hər port ayrı-ayrı (Fa0/1, Fa0/2, Fa0/3, Fa0/4) yazılacaq.
  • CLI ayrımı — Router əmrləri və Switch əmrləri heç vaxt eyni komanda blokunda qarışmayacaq, hər cihaz üçün ayrıca pre/copy bloku olacaq.
  • ICMP ACL nümunəsi — bir PC-nin ping ata bilməsi üçün permit icmp 192.168.10.0 0.0.0.255 host 192.168.10.254 echo formatı əsas nümunə kimi istifadə olunacaq.
  • Copy düyməsi — hər bir komanda bloku üçün ayrıca, işləyən "⧉ Copy" düyməsi olacaq.

Hazır Prompt

Sən Cisco Packet Tracer və şəbəkə konfiqurasiyası üzrə ekspertsən. Sənə əvvəlki iki HTML faylını (cisco_imtahan.html və cisco_imtahan_v2.html) nümunə olaraq atıram. Hər ikisini AŞAĞIDAKİ düzəlişlərlə YENİDƏN yaz (struktur, input cədvəli, IP hesablama məntiqi və dizayn eyni qalsın, yalnız bu problemləri fix et):

1) PORT ARALIQLARI YASAQDIR.
   "FastEthernet0/1-4" və ya "interface range FastEthernet0/1-4" kimi defisli aralıq YAZILMASIN.
   Əvəzinə hər port ayrı-ayrı, tam adı ilə yazılsın:
     interface FastEthernet0/1
      ...komandalar...
      exit
     interface FastEthernet0/2
      ...komandalar...
      exit
   (və s. hər port üçün ayrıca blok). expandRange()/"interface range" funksiyalarından İSTİFADƏ ETMƏ — JS-də portları array kimi tək-tək render et, "interface range" sintaksisini heç yerdə işlətmə.

2) ROUTER VƏ SWITCH KOMANDALARI HEÇ VAXT EYNİ BLOKDA QARIŞMASIN.
   Əvvəlki fayllarda bəzi step-lərdə Router və Switch əmrləri bir pre/cmd blokunda ardıcıl yazılıb (məs. NAT inside Router-də, sonra elə həmin blokda Switch əmri). Bunu fix et:
   - Hər step-in "📍 Harada" sətri YALNIZ bir cihaz/bir rejim göstərsin (məs. "R1 CLI-də, global config" və ya "Sw1 CLI-də, interface Fa0/1").
   - Əgər bir tapşırıq həm Router, həm Switch tərəfində əmr tələb edirsə, bunu İKİ AYRI step-ə böl: biri "Router CLI-də" başlığı ilə öz pre/copy-button blokunda, digəri "Switch CLI-də" başlığı ilə tamam ayrı pre/copy-button blokunda. Heç bir vəziyyətdə bir 
 daxilində həm router, həm switch əmri olmasın.

3) ICMP ÜÇÜN STANDART NÜMUNƏ FORMAT.
   Bir konkret PC-nin Router/Switch-i ping edə bilməsini təmin edən ACL qaydalarında məhz bu formatdan istifadə et (network + wildcard + host + echo):
     access-list  permit icmp <ŞƏBƏKƏ>  host  echo
   Nümunə: permit icmp 192.168.10.0 0.0.0.255 host 192.168.10.254 echo
   Extended ACL-lərdə "echo" açar sözünü unutma (yalnız ping-echo-request icazəsi, echo-reply üçün ayrıca "echo-reply" əlavə edilə bilər, amma defolt olaraq "echo" kifayətdir, lazım gələrsə hər iki istiqaməti DÜŞÜN və izah et).

4) HƏR KOMANDA BLOKUNUN BAŞLIĞI VƏ COPY DÜYMƏSİ.
   Hər bir 
 blokunun üstündə:
   - Hansı cihaz/rejim olduğunu göstərən kiçik başlıq (məs. "R1 — global config" və ya "Sw1 — Fa0/1")
   - Sağ yuxarı küncdə işləyən "⧉ Copy" düyməsi (clipboard.writeText, pre.textContent-i kopyalasın, kopyalandıqdan sonra "✓ Kopyalandı" göstərsin və 1.4 saniyə sonra əvvəlki mətnə qayıtsın)
   olmalıdır — bu artıq əvvəlki fayllarda var, sadəcə indi HƏR AYRI router/switch blokunun da öz başlığı və öz copy düyməsi olduğundan əmin ol (3-cü bənddə bölünən step-lərin hər ikisində ayrıca düymə).

5) DİGƏR HƏR ŞEY EYNİ QALSIN:
   - Yuxarıda "Dəyişə Bilən Parametrlər" cədvəli (interfeys adları, hostname, subnet, port-security, SSH, ACL nömrəsi və s. input/dropdown).
   - IP-lər avtomatik hesablanmalıdır (network+mask əsasında 1-ci/2-ci/3-cü/4-cü/sonuncu usable) — əl ilə IP yazılmamalıdır.
   - Tünd fon, yüksək kontrast, monospace komandalar, mobil-responsive dizayn.
   - Hər step: nömrə+başlıq, 📍 Harada (bir cihaz/bir rejim), tam CLI əmrləri (kod blokunda, dəyişən dəyərlər fərqli rənglə), yalnız zəruri olduqda 1 cümləlik xəbərdarlıq. Uzun "niyə vacibdir" izahları, dərslik mətni YOX.
   - Sonda yalnız ping/show əmrləri olan qısa "Yoxlama" bölməsi.

Hər iki faylı (cisco_imtahan.html və cisco_imtahan_v2.html) bu qaydalarla TAM YENİDƏN yaz və hər birini ayrıca tam self-contained HTML kodu kimi ver.
